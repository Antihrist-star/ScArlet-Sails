DataService API

The DataService module provides read-only access to Scarlet Sails repository data via public GitHub URLs.

Configuration

DataService automatically configures itself with:

{
  REPO_OWNER: 'Antihrist-star',
  REPO_NAME: 'ScArlet-Sails',
  BRANCH: 'main',
  CACHE_TTL: 5 minutes
}

Core Methods

fetchText(path, fallback)

Fetch raw text from repository file.

await DataService.fetchText('/README.md', '-- (No data)')
fetchJSON(path, fallback)

Fetch and parse JSON file from repository.

await DataService.fetchJSON('/results/latest_result.json', {})
getSharpeRatio()

Get latest Sharpe Ratio metric.

const ratio = await DataService.getSharpeRatio();
getWinRate()

Get latest Win Rate metric.

const rate = await DataService.getWinRate();
getTotalTrades()

Get total trade count.

const trades = await DataService.getTotalTrades();
listModels()

Discover available models in repository.

const models = await DataService.listModels();
getModelCode(modelName)

Get model source code.

const code = await DataService.getModelCode('P_rb');
getCacheStats()

Get cache statistics for debugging.

const stats = DataService.getCacheStats();

Data Binding Principles

  • Read-Only: All operations are read-only public data access
  • Zero Secrets: No API keys, tokens, or credentials in code
  • Graceful Fallbacks: Missing data displays "-- (No data)" instead of errors
  • Smart Caching: 5-minute TTL reduces GitHub rate-limit pressure
  • Public URLs Only: Uses public GitHub raw content endpoints
  • No Fabrication: Data is never invented; only real repository data is displayed

Error Handling

All errors are caught and converted to fallback values. Users see "-- (No data available)" instead of errors. Console contains detailed logs for debugging.

// Example: If data doesn't exist, the dashboard shows:
Sharpe Ratio: -- (No data)

// NOT a stack trace or HTTP error

Rate Limiting

GitHub's public file access has rate limits. To avoid hitting them:

  • Caching reduces redundant requests
  • Browser caching helps with repeated visits
  • Consider page refresh delays if using multiple tabs

Repository Data Structure

The dashboard expects the main repository to have:

/README.md              - Project documentation
/models/                - Model implementations
  /P_rb.py
  /P_mi.py
  /P_hyb.py
/results/               - Performance results/metrics
  /latest_result.json   (optional)
  /backtest_results.json (optional)
